# lash-oblique

OBLIQ-Bench math runner for Lash RLM.

The first target is the `math` subset only. The runner gives Lash RLM custom
Qdrant-backed retrieval tools and requires a structured submission containing
exactly 100 ranked corpus document IDs.

## Setup

Requires [`uv`](https://docs.astral.sh/uv/) on PATH. The Python scripts use PEP 723
inline metadata, so `uv` resolves and caches their dependencies automatically on
first invocation â€” no `requirements.txt`, no virtualenv to manage.

Start Qdrant in one terminal:

```bash
scripts/start-qdrant.sh
```

Download the math subset and build the Qdrant collection:

```bash
scripts/setup_math.sh --recreate
```

For a quick smoke index, add `--limit 200`.

## Run One Query

```bash
scripts/run_math.sh --query-id q02193 --output .benchmarks/obliq/runs/q02193.json
```

The runner uses the active provider from `~/.lash/config.json`. Override it with
`--provider-id`, `--model`, or `--variant`.

## Tool Surface

The RLM gets six tools:

- `search(queries, mode, limit, candidate_pool)` â€” corpus retrieval; `mode` is
  `"hybrid"` (default, BM25 + dense + optional late fused via RRF), `"bm25"`,
  `"dense"`, or `"late"`.
- `discover_docs(target_query, context_pairs, limit)` â€” example-anchored search
  using positive/negative document pairs.
- `tournament_rerank(query, candidate_pools, top_k)` â€” listwise tournament
  reranker; takes labeled candidate pools, does canonical RRF (k=60) across
  them, caps at 300, then runs the tournament.
- `llm_query`, `spawn_agent`, `list_async_handles` â€” RLM utilities.

The Qdrant collection stores:

- sparse BM25 vectors under `bm25`, generated by Qdrant's `Qdrant/bm25` inference
- dense semantic vectors under `dense`, generated through OpenRouter with `perplexity/pplx-embed-v1-4b`
- optional ColBERT-style multivectors under `late`; pass `--enable-late` to `scripts/setup_math.sh`

The setup script loads `OPENROUTER_API_KEY` from `/home/sam/code/lash/.env` by default.

All ranked retrieval tools return `{ "matches": [...] }` ordered best-first.
Each match includes `rank`, `doc_id`, `score`, `text` (full text inline), and
`metadata`. Search tools default to `limit: 100` and cap at 200. Hybrid mode
uses Qdrant server-side RRF fusion over BM25 and dense prefetches, plus late
prefetches when the collection was built with `--enable-late`.

## Evaluate A Saved Submission

```bash
cargo run -- eval-math \
  --query-id q02193 \
  --submission .benchmarks/obliq/runs/q02193.json
```

Metrics reported: `NDCG@10`, `NDCG@50`, `Recall@10`, `Recall@50`, and
`Recall@100`.
