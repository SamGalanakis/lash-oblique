# lash-oblique

OBLIQ-Bench analogue runner for Lash RLM.

The runner gives Lash RLM custom Qdrant-backed retrieval tools and requires a
structured submission containing exactly 100 ranked corpus document IDs. Tasks
are addressed as `subset/task_id`; bare task IDs are accepted when they are
unique across selected subsets.

## Setup

Requires [`uv`](https://docs.astral.sh/uv/) on PATH. The Python scripts use PEP 723
inline metadata, so `uv` resolves and caches their dependencies automatically on
first invocation — no `requirements.txt`, no virtualenv to manage.

Start Qdrant in one terminal:

```bash
scripts/start-qdrant.sh
```

Download the selected subsets and build the Qdrant collection:

```bash
scripts/setup_math.sh --recreate
```

By default setup indexes `math,writing` into one Qdrant collection with a
`subset` payload filter. Override with `SUBSETS=math` or
`SUBSETS=math,writing`. For a quick smoke index, add `--limit 200`.

## Run Tasks

```bash
scripts/run_math.sh --task-id q02193 --output .benchmarks/obliq/runs/math/q02193.json
cargo run --release -- run-task --subset writing --task-id <writing-task-id>
cargo run --release -- run-batch --subsets math,writing --limit 10
cargo run --release -- run-batch --tasks math/q02193,writing/<writing-task-id>
cargo run --release -- run-batch --subsets rep20
```

The runner uses the active provider from `~/.lash/config.json`. Override it with
`--provider-id`, `--model`, or `--variant`.

Named representative subsets:

- `math-rep10`: `q02193,q01486,q01066,q02834,q01757,q02979,q01298,q00844,q00847,q01488`
- `writing-rep10`: `1,182,306,487,708,951,1727,1987,2105,2213`
- `rep20`: `math-rep10` plus `writing-rep10`

## Tool Surface

The RLM gets seven tools:

- `search(queries, mode, limit, candidate_pool)` — corpus retrieval; `mode` is
  `"hybrid"` (default, BM25 + dense + optional late fused via RRF), `"bm25"`,
  `"dense"`, or `"late"`.
- `discover_docs(target_query, context_pairs, limit)` — example-anchored search
  using positive/negative document pairs.
- `judge_candidates(verifier_predicate, candidate_doc_ids, surface_bait)` —
  calibrates retrieved ids into likely positives, distractors, and unclear cases,
  then suggests surface bait to avoid and follow-up queries.
- `tournament_rerank(query, candidate_pools, top_k)` — listwise tournament
  reranker; takes labeled candidate pools, does canonical RRF (k=60) across
  them, caps at 300, then runs the tournament.
- `llm_query`, `spawn_agent`, `list_async_handles` — RLM utilities.

The Qdrant collection stores:

- sparse BM25 vectors under `bm25`, generated by Qdrant's `Qdrant/bm25` inference
- dense semantic vectors under `dense`, generated through OpenRouter with `perplexity/pplx-embed-v1-4b`
- optional ColBERT-style multivectors under `late`; pass `--enable-late` to `scripts/setup_math.sh`
- a `subset` payload field; every retrieval call filters to the current task's
  subset so mixed-subset runs do not cross-contaminate results

The setup script loads `OPENROUTER_API_KEY` from `/home/sam/code/lash/.env` by default.

All ranked retrieval tools return `{ "matches": [...] }` ordered best-first.
Each match includes `rank`, `doc_id`, `score`, `text` (full text inline), and
`metadata`. Search tools default to `limit: 100` and cap at 200. Hybrid mode
uses Qdrant server-side RRF fusion over BM25 and dense prefetches, plus late
prefetches when the collection was built with `--enable-late`.

## Evaluate A Saved Submission

```bash
cargo run -- eval-task \
  --subset math \
  --task-id q02193 \
  --submission .benchmarks/obliq/runs/math/q02193.json
```

Metrics reported: `NDCG@10`, `NDCG@50`, `Recall@10`, `Recall@50`, and
`Recall@100`.
